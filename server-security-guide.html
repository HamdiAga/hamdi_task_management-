تأمين السيرفر خطوة بخطوة: كيف تحمي تطبيقك من 90% من الهجمات الشائعة

10 يونيو 2026 حمدي فواز أبو حاتم Hosting & Domains

تخيل أن تستيقظ صباحاً لتجد موقعك مخترقاً، بيانات العملاء مسربة، أو أسوأ: ملفاتك مشفرة ببرنامج فدية. معظم الهجمات لا تستهدف موقعك تحديداً، بل تمسح آلاف السيرفرات بحثاً عن ثغرات معروفة. 90% من الهجمات يمكن منعها بإعدادات أمان أساسية. سأشاركك خطوات عملية لتأمين سيرفرك، من واقع تجربتي في حماية متاجر إلكترونية.

1. الجدار الناري (Firewall) – خط الدفاع الأول

أغلق كل المنافذ التي لا تحتاجها. السيرفر العادي يحتاج فقط:

  • المنفذ 22 (SSH) – لكن غيّره إلى منفذ غير قياسي إن أمكن.
  • المنفذ 80 و 443 (HTTP و HTTPS).
  • المنفذ 3306 (MySQL) يجب أن يكون مغلقاً عن العالم، ومفتوحاً فقط لـ localhost.

استخدم UFW (Uncomplicated Firewall) على Ubuntu:

sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
sudo ufw status verbose

2. تأمين SSH – لا تدع أحداً يخمن كلمة مرورك

  • استخدم SSH Keys بدلاً من كلمة المرور: كلمة المرور يمكن تخمينها، لكن مفتاح SSH لا يمكن اختراقه عملياً.
  • غيّر منفذ SSH الافتراضي (22): هذا يمنع 99% من محاولات الاختراق الآلية.
  • عطّل تسجيل الدخول كـ root: استخدم مستخدماً عادياً ثم sudo su.
# /etc/ssh/sshd_config
Port 2222                      # غيّر المنفذ
PermitRootLogin no             # امنع root من SSH مباشرة
PasswordAuthentication no      # اعتمد المفاتيح فقط
PubkeyAuthentication yes

# بعد التعديل:
sudo systemctl restart sshd

3. Fail2Ban – منع هجمات القوة العمياء (Brute Force)

Fail2Ban يراقب ملفات السجلات (logs)، وإذا اكتشف محاولات فاشلة متكررة (مثل 5 محاولات SSH فاشلة من نفس IP)، فإنه يحظر هذا IP تلقائياً لمدة محددة.

sudo apt install fail2ban -y
sudo systemctl enable fail2ban
sudo systemctl start fail2ban

# تأكد من أنه يحمي SSH
sudo fail2ban-client status sshd

4. تحديثات دورية – الثغرات تُكتشف كل يوم

معظم الاختراقات تحدث عبر ثغرات معروفة لها تصحيحات منذ شهور، لكن السيرفر لم يُحدّث. ضع cron job للتحديثات الأمنية التلقائية:

sudo apt update && sudo apt upgrade -y
# للتحديثات الأمنية فقط:
sudo unattended-upgrades

5. تأمين التطبيق نفسه – صلاحيات الملفات

  • لا تجعل أي مجلد بصلاحية 777 (كتابة للجميع). هذا يدمر الأمان.
  • في Laravel، صلاحيات storage/ و bootstrap/cache/ يجب أن تكون 775، وباقي الملفات 644.
  • تأكد من أن .env غير قابل للقراءة من المتصفح. في Nginx، امنع الوصول للملفات المخفية:
location ~ /\. {
    deny all;
    return 404;
}

6. شهادة SSL و CSP Headers

  • استخدم HTTPS دائماً (Cloudflare SSL مجاني).
  • أضف Content Security Policy (CSP) لمنع هجمات XSS:
add_header Content-Security-Policy "default-src 'self'; script-src 'self' 'unsafe-inline' https://cdnjs.cloudflare.com; style-src 'self' 'unsafe-inline'; img-src 'self' data: https:;" always;
  • أضف X-Frame-Options لمنع تضمين موقعك في iframe (هجمات Clickjacking).
  • أضف X-Content-Type-Options: nosniff لمنع المتصفح من تخمين نوع الملف.

7. نسخ احتياطي – طوق النجاة الأخير

حتى لو فشلت كل الدفاعات، النسخ الاحتياطي ينقذك. قم بأتمتة:

  • نسخ يومي لقاعدة البيانات.
  • نسخ أسبوعي لكل الملفات.
  • النسخ يجب أن يخزن خارج السيرفر (Google Drive، S3، أو استضافة أخرى).

حصيلة: هذه الإجراءات السبعة (جدار ناري، SSH آمن، Fail2Ban، تحديثات، صلاحيات، SSL/CSP، نسخ احتياطي) تمنع 90% من الهجمات الشائعة. لا تحتاج لأن تكون خبير أمني لتطبيقها. ساعتان من الإعداد تحمي موقعك لسنوات.

خلاصة

أمان السيرفر ليس رفاهية، بل مسؤولية. كل يوم تتأخر فيه عن تطبيق هذه الخطوات الأساسية هو يوم إضافي يكون فيه سيرفرك هدفاً سهلاً. ابدأ اليوم: أغلق المنافذ، أمّن SSH، فعّل Fail2Ban، وحدّث نظامك. تنام الليل مرتاح البال.
