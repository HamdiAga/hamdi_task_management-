كيف تجعل موقعك صديقاً لـ Core Web Vitals وتجتاز تحديثات غوغل الجديدة

10 يونيو 2026 حمدي فواز أبو حاتم SEO/ASO

غوغل تعلن كل عام عن تحديثات جديدة لخوارزمياتها، لكن منذ 2021، أصبحت Core Web Vitals عامل ترتيب رسمي. إذا كان موقعك لا يجتاز هذه المؤشرات، فأنت تخسر ترتيبك لصالح منافسين أسرع وأكثر استقراراً. في هذا الدليل العملي، سأشرح لك كل مؤشر وكيف تحسّنه من منظور المطور.

1. ما هي Core Web Vitals؟

ثلاثة مؤشرات تقيس تجربة المستخدم الحقيقية على موقعك:

المؤشرماذا يقيسالحد الممتازالحد السيئ
LCP (Largest Contentful Paint)زمن تحميل أكبر عنصر مرئيأقل من 2.5 ثانيةأكثر من 4.0 ثوانٍ
FID (First Input Delay)زمن استجابة المتصفح لأول تفاعلأقل من 100msأكثر من 300ms
CLS (Cumulative Layout Shift)ثبات التخطيط البصريأقل من 0.1أكثر من 0.25

2. تحسين LCP – اجعل موقعك يظهر بسرعة

أكبر عنصر مرئي عادة يكون صورة أو فيديو أو نص كبير. لتحسين LCP:

  • حوّل الصور إلى WebP أو AVIF: هاتان الصيغتان تقللان حجم الصورة حتى 70% بدون خسارة جودة مرئية. استخدم sharp في Node.js أو Intervention Image في Laravel.
  • استخدم CDN: Cloudflare أو AWS CloudFront يوزع صورك من أقرب موقع جغرافي للمستخدم. في الخليج، هذا يعني سيرفرات في الإمارات أو السعودية.
  • فعّل Lazy Loading خارج الشاشة فقط: الصورة التي تظهر فوراً (Above the fold) يجب أن تُحمّل فوراً بدون lazy loading. أضف loading="lazy" فقط للصور التي تظهر بعد التمرير.
  • Preload الصور الحرجة: أضف <link rel="preload" as="image" href="hero.jpg" /> للصورة الرئيسية.

3. تحسين FID – اجعل موقعك يستجيب فوراً

FID يقيس التأخير بين ضغط المستخدم على زر واستجابة المتصفح. السبب الرئيسي: JavaScript الثقيل الذي يحجب الخيط الرئيسي. الحلول:

  • قسّم JavaScript: استخدم Code Splitting (كما شرحنا في مقال سابق). لا تحمّل 2MB من JS في الصفحة الرئيسية.
  • استخدم Web Workers: انقل العمليات الثقيلة (مثل معالجة الصور أو الحسابات) إلى Web Worker لتعمل في الخلفية دون حجب الصفحة.
  • Defer JavaScript: أضف defer أو async لكل السكريبتات غير الضرورية للعرض الأولي.
  • قلل JavaScript الخارجي: كل إضافة أو شريط أدوات طرف ثالث (مثل Facebook Pixel أو Google Maps) تضيف تأخيراً. اسأل نفسك: هل أحتاجها فعلاً في هذه الصفحة؟

4. تحسين CLS – امنع المحتوى من القفز

CLS هو أكثر مؤشر يزعج المستخدمين: تفتح صفحة، تضغط على زر، وفجأة يقفز الزر بسبب تحميل صورة أو إعلان فوقه. الحلول:

  • حدد أبعاد كل الصور والفيديوهات: width="800" height="600" في HTML أو aspect-ratio في CSS. هذا يحجز المساحة قبل تحميل الصورة.
  • لا تدخل محتوى ديناميكياً فوق المحتوى الحالي: إذا أضفت Banner إعلاني، احجز له مساحة مسبقاً.
  • الخطوط المخصصة: استخدم font-display: swap لمنع النص من الاختفاء ثم الظهور بخط مختلف.

5. كيف تقيس Core Web Vitals؟

  • Google Search Console: قسم Core Web Vitals يريك بالضبط أي صفحات تحتاج تحسيناً.
  • PageSpeed Insights: يعطيك تقريراً مفصلاً مع توصيات محددة.
  • Lighthouse: مدمج في Chrome DevTools. شغّله على أي صفحة لترى النتيجة.
  • Chrome User Experience Report (CrUX): بيانات حقيقية من مستخدمين فعليين.

من تجربتي: في أحد متاجر سلة، كان LCP 5.8 ثانية بسبب صور المنتجات بدقة 4000px. بعد تحويلها لـ WebP وإضافة CDN، انخفض LCP إلى 1.4 ثانية، وارتفع ترتيب الموقع 14 مركزاً في أسبوعين.

خلاصة

Core Web Vitals ليست موضة عابرة، بل هي معيار غوغل الجديد لتجربة المستخدم. LCP للسرعة، FID للاستجابة، CLS للثبات. حسّن هذه المؤشرات الثلاثة، وسترى موقعك يرتفع في الترتيب بينما يتراجع المنافسون المتكاسلون.
