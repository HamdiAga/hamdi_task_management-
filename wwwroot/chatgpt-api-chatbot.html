كيف تستخدم ChatGPT API لبناء بوت خدمة عملاء ذكي لموقعك بدون خبرة سابقة

10 يونيو 2026 حمدي فواز أبو حاتم Automation & AI

تخيل أن موقعك يرد على استفسارات العملاء فوراً، 24 ساعة يومياً، 7 أيام أسبوعياً، وبدقة مذهلة. هذا ليس خيالاً علمياً، بل واقع باستخدام ChatGPT API. في هذا الدليل العملي، سأريك خطوة بخطوة كيف تبني بوت خدمة عملاء ذكي لموقعك، حتى لو لم تستخدم ChatGPT API من قبل.

1. لماذا ChatGPT API وليس بوتاً تقليدياً؟

البوتات التقليدية تعتمد على ردود مبرمجة مسبقاً: "إذا قال العميل X، رد بـ Y". هذا يعمل لـ 10 أسئلة، لكن ماذا عن السؤال الحادي عشر؟ ChatGPT يفهم اللغة الطبيعية، يستطيع الرد على أسئلة لم تبرمجها، ويتذكر سياق المحادثة. الفارق يشبه الفرق بين آلة بيع وموظف خدمة عملاء حقيقي.

2. المتطلبات الأساسية

  • مفتاح API من OpenAI (تحتاج حساباً ولو برصيد 5 دولارات يكفي لآلاف المحادثات).
  • سيرفر يدعم Node.js أو PHP أو بايثون (أي لغة تستطيع عمل HTTP Requests).
  • صفحة HTML بسيطة للواجهة.

3. بناء الـ Backend في Node.js

// server.js
const express = require('express');
const cors = require('cors');
const app = express();
app.use(cors());
app.use(express.json());

const OPENAI_API_KEY = 'sk-your-key-here';

app.post('/api/chat', async (req, res) => {
    try {
        const userMessage = req.body.message;
        const conversationHistory = req.body.history || [];

        const response = await fetch('https://api.openai.com/v1/chat/completions', {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json',
                'Authorization': `Bearer ${OPENAI_API_KEY}`
            },
            body: JSON.stringify({
                model: 'gpt-4o-mini',
                messages: [
                    {
                        role: 'system',
                        content: `أنت بوت خدمة عملاء لمتجر إلكتروني اسمه "متجري".
مهامك: الرد على استفسارات العملاء، مساعدتهم في تتبع الطلبات،
وتقديم معلومات عن المنتجات. كن مهذباً ومفيداً.
إذا سألك العميل عن طلب، اطلب منه رقم الطلب.
إذا سألك عن منتج غير موجود، اقترح منتجات بديلة.
تحدث باللغة العربية الفصحى مع لمسة ودية.`
                    },
                    ...conversationHistory,
                    { role: 'user', content: userMessage }
                ],
                temperature: 0.7,
                max_tokens: 300
            })
        });

        const data = await response.json();
        const botReply = data.choices[0].message.content;

        res.json({ reply: botReply });
    } catch (error) {
        console.error('Error:', error);
        res.status(500).json({ reply: 'عذراً، حدث خطأ.' });
    }
});

app.listen(3000, () => console.log('Chatbot API running on port 3000'));

4. الواجهة الأمامية (HTML + JavaScript)

<!-- chatbot.html -->
<div id="chatbot-widget">
    <div id="chat-messages"></div>
    <div id="chat-input-area">
        <input type="text" id="user-input" placeholder="اكتب سؤالك هنا..." />
        <button onclick="sendMessage()">إرسال</button>
    </div>
</div>

<script>
let conversationHistory = [];

async function sendMessage() {
    const input = document.getElementById('user-input');
    const message = input.value.trim();
    if (!message) return;

    // أضف رسالة المستخدم للواجهة
    addMessage('user', message);
    input.value = '';

    // أرسل إلى الـ Backend
    const response = await fetch('https://api.yoursite.com/api/chat', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ message, history: conversationHistory })
    });

    const data = await response.json();
    addMessage('bot', data.reply);

    // حدّث تاريخ المحادثة
    conversationHistory.push({ role: 'user', content: message });
    conversationHistory.push({ role: 'assistant', content: data.reply });
}

function addMessage(sender, text) {
    const messagesDiv = document.getElementById('chat-messages');
    const msgDiv = document.createElement('div');
    msgDiv.className = sender === 'user' ? 'user-message' : 'bot-message';
    msgDiv.textContent = text;
    messagesDiv.appendChild(msgDiv);
    messagesDiv.scrollTop = messagesDiv.scrollHeight;
}
</script>

5. ربط البوت بقاعدة بيانات المتجر (للاستفسارات الحقيقية)

الخطوة السابقة تعطي بوتاً يتحدث بشكل عام. لكن ماذا لو سأل العميل "أين طلبي رقم 12345"؟ هنا تحتاج Function Calling:

// أضف هذه الدالة في الـ Backend قبل استدعاء ChatGPT
async function getOrderStatus(orderId) {
    const order = await db.query('SELECT status FROM orders WHERE id = ?', [orderId]);
    return order ? `حالة طلبك رقم ${orderId}: ${order.status}` : 'لم أجد طلباً بهذا الرقم.';
}

// في الرسالة للنظام (system prompt)، أضف:
"إذا سألك العميل عن طلب، استخدم رقم الطلب للبحث في قاعدة البيانات.
قاعدة البيانات تحتوي على طلبات حقيقية. رقم الطلب يكون رقماً صحيحاً."

لتطبيق Function Calling بشكل كامل، استخدم ميزة tools في OpenAI API لتخبر ChatGPT عن الدوال المتاحة، وهو سيقرر متى يستدعيها.

6. تكاليف التشغيل

باستخدام GPT-4o-mini، تكلفة 1000 محادثة قصيرة (حوالي 50 سنتاً). هذا يعني أن 5 دولارات تكفي لشهر كامل من الخدمة لمتجر متوسط. التكلفة أقل بكثير من توظيف موظف خدمة عملاء.

أمان: لا ترسل بيانات حساسة (أرقام بطاقات، كلمات مرور) إلى OpenAI API. استخدم نظاماً هجيناً: البوت يجيب على الأسئلة العامة، ويحول الأسئلة الحساسة لموظف بشري.

خلاصة

ChatGPT API حوّل بناء بوتات ذكية من مشروع معقد يحتاج فريق AI إلى مهمة يمكن لمطور واحد إنجازها في يوم. ابدأ بالـ system prompt الجيد، اربطه بقاعدة بياناتك، وراقب المحادثات لتحسينه باستمرار. عملاؤك سيقدرون الردود الفورية، وستوفر ساعات من خدمة العملاء اليدوية.
