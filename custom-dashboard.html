بناء واجهة إدارة متجر كاملة (Dashboard) بدلاً من الاعتماد على إضافات جاهزة

9 يونيو 2026 حمدي فواز أبو حاتم Developer Full-Stack

لوحة التحكم هي مركز قيادة المتجر. لكن الإضافات الجاهزة (مثل WooCommerce Admin أو إضافات سلة) غالباً ما تكون إما محدودة أو مثقلة بميزات لا تحتاجها. في هذا المقال، سأشرح متى تحتاج لبناء لوحة تحكم مخصصة، وكيف تبنيها بسرعة مع Laravel + Blade أو Vue، مع أمثلة من واقع المتاجر السعودية.

1. متى تبني Dashboard مخصصة؟

ليس كل متجر يحتاج لوحة تحكم مخصصة. اسأل نفسك:

  • هل تحتاج تقارير محددة غير متوفرة في الإضافات الجاهزة؟ (مثلاً: تقرير مبيعات كل مندوب على حدة، تقرير ضريبة القيمة المضافة، ...).
  • هل تريد دمج بيانات من مصادر متعددة في شاشة واحدة؟ (مثلاً: طلبات سلة + طلبات واتساب + طلبات الهاتف).
  • هل تحتاج صلاحيات دقيقة جداً لا توفرها الإضافات؟
  • هل الإضافة الجاهزة تبطئ موقعك؟

إذا أجبت بنعم على أي من هذه، فأنت تحتاج Dashboard مخصصة.

2. التقنية المناسبة

الطريقةالمميزاتالسيئات
Laravel + Bladeسريع، بسيط، لا تحتاج API منفصلأقل تفاعلية، إعادة تحميل للصفحة
Laravel API + Vue/Reactتفاعلية عالية، SPA سريعةيحتاج API منفصل، وقت أطول للتطوير
Laravel + Livewireتفاعلية بدون كتابة JavaScript كثيرأداء أقل في الصفحات المعقدة جداً

نصيحتي: ابدأ بـ Blade + Livewire لمعظم المشاريع. إذا كبر المشروع، انقل تدريجياً إلى Vue/React مع API.

3. هيكل الـ Dashboard

لوحة التحكم المخصصة عادة تحتوي على:

  • Layout ثابت: Sidebar للتنقل، Header مع اسم المستخدم وإشعارات.
  • شاشات: Dashboard رئيسية (إحصائيات)، إدارة المنتجات، الطلبات، العملاء، التقارير.
  • Widgets: بطاقات أرقام (مبيعات اليوم، طلبات معلقة، منتجات منخفضة المخزون).

4. مثال: بناء Widget المبيعات السريعة في Laravel Blade

// app/Http/Controllers/DashboardController.php
public function index() {
    return view('dashboard.index', [
        'todaySales' => Order::whereDate('created_at', today())->sum('total'),
        'pendingOrders' => Order::where('status', 'pending')->count(),
        'lowStockProducts' => Product::where('stock', '<', 5)->count(),
        'monthlyRevenue' => Order::whereMonth('created_at', now()->month)->sum('total'),
    ]);
}

في الواجهة (Blade):

<!-- resources/views/dashboard/index.blade.php -->
<div class="grid grid-cols-4 gap-4">
    <x-stat-card title="مبيعات اليوم" :value="$todaySales" icon="ri-money-dollar-circle-line" color="green" />
    <x-stat-card title="طلبات معلقة" :value="$pendingOrders" icon="ri-time-line" color="orange" />
    <x-stat-card title="منتجات منخفضة المخزون" :value="$lowStockProducts" icon="ri-alert-line" color="red" />
    <x-stat-card title="إيرادات الشهر" :value="$monthlyRevenue" icon="ri-line-chart-line" color="blue" />
</div>

5. مثال من السوق – لوحة تحكم لمتجر سلة

في أحد مشاريعي، كان صاحب متجر سلة يشتكي من أن تقارير سلة لا تظهر له "أداء كل مندوب توصيل" و"المنتجات الأكثر إرجاعاً". بنينا Dashboard مخصصة تستدعي بيانات سلة عبر API الخاص بهم، وتضيف هذين التقريرين. النتيجة: وفرنا على صاحب المتجر 3 ساعات أسبوعياً كان يقضيها في تجميع هذه البيانات يدوياً من Excel.

6. نصائح سريعة

  • لا تبالغ في البناء: ابدأ بأهم 3 شاشات فقط. أضف المزيد بناءً على احتياج العميل.
  • استخدم Charts: Chart.js أو ApexCharts لرسم المبيعات الشهرية، توزيع الطلبات حسب المناطق، إلخ.
  • اجعلها Responsive: صاحب المتجر سيفتح اللوحة من جواله كثيراً.
  • أضف Export: أي جدول يجب أن يكون له زر "تصدير Excel".

خلاصة

لوحة التحكم المخصصة تستحق وقتك إذا كانت الإضافات الجاهزة لا تلبي احتياجات العميل. ابدأ صغيراً، استخدم Blade + Livewire، وأضف الميزات تدريجياً. العميل سيرى قيمة عملك فوراً عندما يجد التقرير الذي يحتاجه في ثانية واحدة بدلاً من ساعة.
