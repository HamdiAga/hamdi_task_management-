ربط أنظمة المدفوعات الإلكترونية في السعودية والخليج: دليل المطوّر العملي

9 يونيو 2026 حمدي فواز أبو حاتم Developer Full-Stack

الدفع الإلكتروني في الخليج يختلف تماماً عن الأسواق الغربية. لدينا مدى، تمارا، STC Pay، Tap، وغيرها. كمطوّر، ربط هذه البوابات بشكل صحيح هو الفرق بين متجر يبيع ومتجر يخسر عملاءه عند صفحة الدفع. سأشرح لك كيف تتعامل مع كل بوابة، مع أمثلة عملية ونصائح من واقع السوق.

2. نموذج ربط بوابة مدى (HyperPay كمثال)

معظم بوابات الدفع السعودية تتطلب تدفقاً مماثلاً:

  1. أنشئ طلب دفع (Checkout) عبر API برقم طلب فريد وسعر ومصادقة.
  2. استلم رابط أو redirect_token من البوابة.
  3. وجّه المستخدم إلى صفحة الدفع الآمنة على البوابة.
  4. بعد الدفع، البوابة ترسل Webhook إلى عنوان خاص في API تطبيقك (مثل /api/payment/callback).
  5. تحقق من صحة الـ Webhook باستخدام Secret Key، ثم حدّث حالة الطلب.
// Laravel: معالجة Webhook من بوابة الدفع
public function handleWebhook(Request $request) {
    $payload = $request->all();
    $signature = $request->header('X-Signature');

    // تحقق من التوقيع
    if (!$this->isValidSignature($payload, $signature)) {
        return response()->json(['error' => 'Invalid signature'], 401);
    }

    $order = Order::where('payment_id', $payload['transaction_id'])->first();
    if ($order && $payload['status'] === 'success') {
        $order->update(['status' => 'paid', 'paid_at' => now()]);
        // أرسل إشعار للعميل
    }

    return response()->json(['success' => true]);
}

3. الدفع لاحقاً – تمارا وTabby

الدفع بالتقسيط والدفع لاحقاً أصبح مهماً جداً في السوق السعودي. طريقة الربط:

  • أرسل تفاصيل الطلب (المنتجات، الأسعار، معلومات العميل) إلى API تمارا.
  • تمارا ترجع redirect_url لصفحة الدفع الخاصة بهم.
  • بعد موافقة تمارا، تستلم Webhook وتحدث حالة الطلب.

4. ربط بوابات الدفع مع WooCommerce وسلة وشوبفاي

  • WooCommerce: استخدم إضافات جاهزة لـ Tap أو مدى. لكن إن كنت تريد تخصيصاً عميقاً، استخدم WooCommerce Payment Gateway API وابنِ تكاملك الخاص.
  • سلة: سلة توفر API جاهز للدفع مع مدى وتمارا. دورك كمطوّر هو تنسيق تجربة المستخدم بعد الدفع (صفحة شكر مخصصة، إشعارات).
  • شوبفاي: استخدم Shopify Payments أو إضافات البوابات المحلية. شوبفاي يدعم معظم البوابات الخليجية عبر إضافات الطرف الثالث.

تحذير من تجربتي: في أحد المشاريع، نسينا توثيق Webhook بشكل صحيح، فتمكن مخترق من إرسال Webhooks مزيفة وتغيير حالة الطلبات إلى "مدفوع" دون دفع فعلي. استخدم دائماً التحقق من التوقيع (HMAC) ولا تثق بأي Webhook بدون مصادقة.

5. أفضل الممارسات عند ربط المدفوعات

  • Idempotency Keys: أرسل مفتاحاً فريداً مع كل طلب دفع لمنع تكرار العملية إذا ضغط المستخدم مرتين.
  • تسجيل كل شيء: سجّل كل طلب وكل Webhook وكل خطأ في جدول payment_logs. هذا ينقذك عند حدوث نزاع مع العميل.
  • تجربة السقوط (Fallback): إذا فشلت بوابة الدفع، اعرض للمستخدم خياراً آخر. لا تخسر عملية بيع بسبب تعطل بوابة.
  • الامتثال لـ PCI DSS: لا تخزّن أبداً بيانات بطاقات الائتمان في قاعدة بياناتك. استخدم tokens من البوابة نفسها.

خلاصة

الدفع الإلكتروني في الخليج سوق ديناميكي. كمطوّر، فهمك لهذه البوابات وطريقة ربطها بشكل آمن وسلس يجعلك شريكاً تقنياً لا غنى عنه للتجار. ابدأ ببوابة واحدة، أتقنها، ثم وسّع.
