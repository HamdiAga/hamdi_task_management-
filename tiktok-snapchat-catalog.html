ربط المتجر بالتيكتوك وسناب شات كتالوج: حلول لمشاكل "المنتجات مرفوضة"

10 يونيو 2026 حمدي فواز أبو حاتم Technical Marketing

تيكتوك وسناب شات لم يعودا مجرد تطبيقات ترفيه، بل أصبحا قنوات بيع قوية. لكن ربط كتالوج منتجات متجرك بهما قد يكون كابوساً تقنياً: منتجات مرفوضة، بيانات غير متزامنة، وأخطاء غامضة. بعد معالجة عشرات حالات الرفض لمتاجر سلة وزِد وشوبفاي، سأشاركك الأسباب الحقيقية لرفض المنتجات وحلولها التقنية.

1. كيف يعمل كتالوج تيكتوك وسناب شات؟

المنصتان تقدمان ميزة "كتالوج المنتجات" أو Product Catalog، وهي عبارة عن تغذية بيانات (Data Feed) بتنسيق XML أو CSV أو عبر API مباشر. المنصة تقرأ هذا الملف يومياً، وتحدث قائمة منتجاتك في متجرك داخل التطبيق. أي خطأ في هذا الملف يؤدي إلى رفض المنتج.

2. أشهر أسباب رفض المنتجات وحلولها

3. إنشاء Feed متوافق لسلة وشوبفاي

لمستخدمي سلة: سلة تولد Feed تلقائياً يمكنك استخدامه. تأكد من أن المنتجات تحتوي على: اسم المنتج، وصف، سعر، SKU، صورة رئيسية بخلفية بيضاء، ورابط المنتج.

لمستخدمي شوبفاي: استخدم Shopify's Facebook & Instagram channel الذي يولد Feed متوافقاً مع تيكتوك أيضاً. لكن راجع feed بنفسك: اذهب إلى Products > Export، وتأكد من الأعمدة المطلوبة.

للمتاجر المخصصة (Laravel/Node.js): ابنِ API endpoint يولد XML Feed تلقائياً:

// Laravel: routes/web.php
Route::get('/tiktok-feed.xml', function() {
    $products = Product::with('images')->where('status', 'active')->get();
    
    $xml = new SimpleXMLElement('');
    
    foreach ($products as $product) {
        $item = $xml->channel->addChild('item');
        $item->addChild('g:id', $product->sku);
        $item->addChild('g:title', htmlspecialchars($product->name));
        $item->addChild('g:description', htmlspecialchars(strip_tags($product->description)));
        $item->addChild('g:link', route('products.show', $product));
        $item->addChild('g:image_link', asset('storage/' . $product->images->first()->path));
        $item->addChild('g:price', number_format($product->price, 2) . ' SAR');
        $item->addChild('g:availability', $product->stock > 0 ? 'in_stock' : 'out_of_stock');
        if ($product->brand) {
            $item->addChild('g:brand', htmlspecialchars($product->brand));
        }
        $item->addChild('g:condition', 'new');
    }
    
    return response($xml->asXML(), 200)->header('Content-Type', 'application/xml');
});

4. أتمتة تحديث الـ Feed

الـ Feed يجب أن يتحدث يومياً. ضع cron job ينشئ الملف كل صباح:

0 3 * * * php /path/to/artisan feed:generate-tiktok

ثم تأكد من أن الرابط الذي أعطيته لتيكتوك (مثل https://yoursite.com/tiktok-feed.xml) يعمل ويعرض البيانات الصحيحة.

5. فحص الـ Feed قبل الإرسال

قبل إرسال الـ Feed للمنصة، افحصه باستخدام:

  • TikTok Ads Manager > Assets > Catalogs: حمّل الـ Feed يدوياً وشاهد تقرير الأخطاء.
  • Google Merchant Center: حتى لو كنت تستهدف تيكتوك، ارفع الـ Feed لـ Google Merchant Center مؤقتاً لتحصل على تقرير أخطاء مفصل ومجاني.

خطأ شائع: كثير من المتاجر تنسى تحديث شهادة SSL، فيفشل تيكتوك في جلب الصور لأنها على HTTP. تأكد أن كل روابط الصور تبدأ بـ https://.

خلاصة

ربط كتالوج منتجاتك بتيكتوك وسناب شات ليس سحراً، بل هو عملية تقنية منظمة: صور نظيفة، بيانات كاملة، Feed محدث يومياً، وفحص دوري للأخطاء. عالج هذه النقاط الأربع، وستلاحظ الفرق في أداء إعلاناتك.
