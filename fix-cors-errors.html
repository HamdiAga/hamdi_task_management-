أفضل ممارسات ربط الـ Frontend بالخلفية وتجنب أخطاء الـ CORS نهائياً

9 يونيو 2026 حمدي فواز أبو حاتم Developer Full-Stack

خطأ CORS هو أشهر خطأ يواجهه مطورو الويب عند ربط Frontend بـ Backend. "Access to fetch at 'https://api.example.com' from origin 'https://frontend.example.com' has been blocked by CORS policy" – جملة أرعبت كل مطور. في هذا المقال، سأشرح لك ما هو CORS بالضبط، ولماذا يوجد، وكيف تتجنبه باحترافية، مع سيناريوهات حقيقية من مشاريع React + Laravel API.

1. ما هو CORS ولماذا هو موجود؟

CORS اختصار لـ Cross-Origin Resource Sharing. هو آلية أمان في المتصفحات تمنع موقعاً (Origin) من عمل طلبات HTTP إلى Origin آخر، إلا إذا سمح الـ Origin الآخر بذلك صراحة.

مثال: إذا كان Frontend على https://myshop.com وحاول الاتصال بـ API على https://api.myshop.com، فالمتصفح سيمنع الطلب لأن الـ Origin مختلف (subdomain مختلف). الحل هو أن يرسل السيرفر (الـ API) هيدرات خاصة تسمح لـ Frontend بالوصول.

2. الحل في Laravel

في Laravel، استخدم Middleware مخصص أو حزمة fruitcake/laravel-cors (أو الإعداد المدمج في Laravel 11):

// config/cors.php
return [
    'paths' => ['api/*'],
    'allowed_methods' => ['*'],
    'allowed_origins' => ['https://myshop.com'],
    'allowed_origins_patterns' => [],
    'allowed_headers' => ['*'],
    'exposed_headers' => [],
    'max_age' => 0,
    'supports_credentials' => true,
];

3. الحل في Node.js (Express)

const cors = require('cors');
app.use(cors({
    origin: 'https://myshop.com',
    methods: ['GET', 'POST', 'PUT', 'DELETE'],
    credentials: true
}));

4. سيناريو: React على Vercel و Laravel API على استضافة مشتركة

في أحد مشاريعي، كان الـ Frontend على Vercel والـ Backend على MonsterASP. ظهر خطأ CORS. الأسباب:

  • نسيت إضافة origin الـ Frontend في إعدادات CORS.
  • الـ Frontend كان يرسل طلبات بدون credentials، بينما الـ API يتطلب cookies.

الحل: تأكد أن Frontend يرسل credentials: 'include' مع كل طلب fetch/axios، وأن السيرفر يسمح بـ credentials: true.

5. سيناريو: طلبات Preflight (OPTIONS)

عندما يرسل المتصفح طلباً مع هيدرات مخصصة (مثل Authorization) أو methods غير GET/POST، يرسل أولاً طلباً تمهيدياً (Preflight) من نوع OPTIONS. تأكد أن سيرفرك يستجيب لـ OPTIONS بشكل صحيح:

// Laravel: في routes/api.php أو Middleware
Route::options('{any}', function() {
    return response('OK', 200);
})->where('any', '.*');

6. CORS في بيئة التطوير المحلية

إذا كنت تعمل محلياً (localhost:3000 لـ React و localhost:8000 لـ Laravel)، فستواجه CORS أيضاً. أسرع حل:

  • في Laravel: أضف 'allowed_origins' => ['http://localhost:3000'].
  • في React (Vite): استخدم proxy في vite.config.js ليتم توجيه طلبات /api إلى Laravel.

نصيحة ذهبية: لا تستخدم '*' في allowed_origins في بيئة الإنتاج أبداً. هذا يفتح API الخاص بك لأي موقع في العالم. حدد الـ origins المسموحة بدقة.

7. مشاكل أخرى قد تخلطها مع CORS

  • خطأ 404 أو 500: أحياناً المتصفح يظهر خطأ CORS، والمشكلة الحقيقية أن الـ API يرجع 404 أو 500. تحقق من Network Tab أولاً.
  • Mixed Content: إذا كان Frontend على HTTPS والـ API على HTTP، المتصفح سيمنع الطلب. استخدم HTTPS دائماً.
  • جدار الحماية أو Cloudflare: تأكد أن إعدادات Cloudflare أو سيرفر الاستضافة لا تمنع هيدرات CORS.

خلاصة

CORS ليس خطأ برمجياً، بل سياسة أمان. تعلم كيف تضبطه بشكل صحيح، وستوفر على نفسك ساعات من الحيرة. تذكر: حدد الـ origins، اسمح بـ credentials فقط إذا كنت تحتاجها، ولا تستخدم wildcard في الإنتاج.
