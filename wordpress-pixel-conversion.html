تتبع التحويلات على ووردبريس للمبتدئين: دليل المبرمج لتركيب البيكسل بدون أخطاء

10 يونيو 2026 حمدي فواز أبو حاتم Technical Marketing

تتبع التحويلات هو الفرق بين إعلانات ناجحة وإعلانات تحرق فلوسك. إذا كنت تدفع على إعلانات فيسبوك أو غوغل ولا تعرف بالضبط أي إعلان جلب لك عملية بيع، فأنت تصب المال في حفرة. سأشرح لك كيف تركب Facebook Pixel و Google Tag Manager على ووردبريس كمبرمج محترف، مع التركيز على Server-Side Tracking لتجنب أخطاء المتصفحات و Ad Blockers.

1. لماذا Server-Side Tracking وليس مجرد Pixel في الـ Frontend؟

طريقة الـ Frontend التقليدية (لصق كود البيكسل في header.php) تعاني من مشكلتين:

  • Ad Blockers: حوالي 30-40% من المستخدمين لديهم مانع إعلانات يمنع Facebook Pixel من التحميل.
  • iOS 14+ و ITP: متصفحات آبل تمنع تتبع الطرف الثالث، مما يقلل دقة البيانات.

الـ Server-Side Tracking يرسل الأحداث من سيرفرك مباشرة إلى فيسبوك، متجاوزاً مانعات الإعلانات وقيود المتصفحات. الدقة ترتفع من 60% إلى 95%+.

2. تركيب Facebook Pixel عبر Server-Side (Conversions API)

الخطوات العملية:

  1. في Facebook Events Manager، أنشئ Pixel وانسخ Pixel ID.
  2. أنشئ Access Token من إعدادات Conversions API.
  3. في ووردبريس، أضف الكود التالي في functions.php أو في إضافة مخصصة:
// في functions.php أو في إضافة مخصصة
add_action('woocommerce_thankyou', 'send_purchase_event_to_facebook', 10, 1);

function send_purchase_event_to_facebook($order_id) {
    $order = wc_get_order($order_id);
    
    $data = [
        'data' => [[
            'event_name' => 'Purchase',
            'event_time' => time(),
            'action_source' => 'website',
            'event_id' => 'order_' . $order_id,
            'user_data' => [
                'em' => [hash('sha256', $order->get_billing_email())],
                'ph' => [hash('sha256', $order->get_billing_phone())],
                'fn' => [hash('sha256', $order->get_billing_first_name())],
                'ln' => [hash('sha256', $order->get_billing_last_name())],
                'client_ip_address' => $_SERVER['REMOTE_ADDR'],
                'client_user_agent' => $_SERVER['HTTP_USER_AGENT']
            ],
            'custom_data' => [
                'currency' => 'SAR',
                'value' => $order->get_total(),
                'content_type' => 'product',
                'contents' => get_order_items_data($order)
            ]
        ]],
        'access_token' => 'YOUR_ACCESS_TOKEN'
    ];

    wp_remote_post(
        "https://graph.facebook.com/v21.0/YOUR_PIXEL_ID/events",
        [
            'body' => json_encode($data),
            'headers' => ['Content-Type' => 'application/json']
        ]
    );
}

function get_order_items_data($order) {
    $items = [];
    foreach ($order->get_items() as $item) {
        $product = $item->get_product();
        $items[] = [
            'id' => $product->get_sku() ?: $product->get_id(),
            'quantity' => $item->get_quantity(),
            'item_price' => $item->get_total() / $item->get_quantity()
        ];
    }
    return $items;
}

3. تركيب Google Tag Manager (GTM) بشكل صحيح

GTM يسمح لك بإدارة كل بيكسلات التسويق من مكان واحد دون تعديل كود الموقع كل مرة. لتركيبه:

  1. أنشئ حساباً على tagmanager.google.com.
  2. انسخ الكودين (واحد لـ head والثاني لـ body).
  3. في قالب ووردبريس، أضف الكود الأول مباشرة بعد فتح <head>، والثاني مباشرة بعد فتح <body>.

للطريقة البرمجية عبر functions.php:

add_action('wp_head', function() {
    echo "
    
    ";
});

add_action('wp_body_open', function() {
    echo '
    
    ';
});

4. التحقق من صحة التتبع

بعد التركيب، لا تثق بأن كل شيء يعمل. اختبر:

  • Facebook Pixel Helper: إضافة Chrome تريك إذا كان البيكسل يعمل والأحداث التي ترسل.
  • Google Tag Assistant: إضافة Chrome لفحص GTM والوسوم.
  • Facebook Events Manager > Test Events: يريك الأحداث التي تصل من سيرفرك مباشرة.

5. أخطاء شائعة يجب تجنبها

  • ازدواجية الأحداث: إذا استخدمت Pixel في الـ Frontend وأيضاً Conversions API، قد تتضاعف الأحداث. استخدم نفس event_id في الحالتين ليمنع فيسبوك التكرار.
  • نسيان حذف بيانات الاختبار: تأكد من تعطيل وضع الاختبار في Facebook Pixel قبل تشغيل الإعلانات الحقيقية.
  • عدم تتبع كل مسار التحويل: لا تكتفِ بـ Purchase. أضف ViewContent، AddToCart، InitiateCheckout لتعرف أين ينسحب العميل.

النتيجة: بعد تطبيق Server-Side Tracking على أحد متاجر ووردبريس، ارتفعت دقة تتبع التحويلات من 62% إلى 96%. هذا يعني أن فيسبوك صار يعرف بالضبط أي الإعلانات تجلب مبيعات، فانخفضت تكلفة النقرة 25%.

خلاصة

تتبع التحويلات ليس مجرد لصق كود. Server-Side Tracking عبر Conversions API يمنحك دقة أعلى، وGTM يمنحك مرونة إدارة كل البيكسلات. استثمر ساعتين في تركيبهما بشكل صحيح، وستوفر آلاف الريالات على الإعلانات المهدرة.
